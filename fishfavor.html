Fishfavor

Non-technical details

There are some basic components that are needed in order to make the system functional. The following is a list of them in descending order of complexity:

  • [ ] GPX importer: The component that handles the GPX files that Trackwell sends. It will take a GPX file as an input and create an entry in the database corresponding to that fishing trip. It may or may not create QR codes as well - is that even possible?

  • [ ] QR code explorer: This component provides fisheries/admins with a QR code explorer where they can browse boats and trips and get a list of QR codes for each of the species that was fished in a particular trip.

  • [ ] QR Viewer: The consumer-facing site where people go when they actually scan a QR code in the store. Should obviously be optimized for mobile browsers.

  • [ ] The main page: The page that will be displayed when we enter http://www.fishflavor.org. This one is somewhat ready but still needs the main section, i.e. the "Find my fish" section.

Technical details

Public endpoints needed

  • http://www.fishfavor.org/c/:qr - for the retrieval of data regarding some fish.
  • http://www.fishfavor.org/ - the main site.

Private (authenticated) endpoints needed

  • http://www.fishfavor.org/data/gpx-importer - an Trackwell GPX importer.
  • http://www.fishfavor.org/manage/ - an endpoint where authenticated users can browse, view and print QR codes.

Data model

  • Trip
    • start_harbor:integer -> foreign key reference to the "start" harbor.
    • landing_harbor:integer -> foreign key reference to the delivery harbor.
    • landed_at:datetime -> landing time.
    • track:text -> an array of maps containing the following keys: 'lat', 'lon', 'temperature', 'elevation'
    • boat:integer -> foreign key reference to some boat.
    • gear:string -> the fishing gear that was used in this trip.
  • Harbor
    • description:text -> static metadata describing the harbor.
  • Boat
    • name:string -> the name of the boat.
    • description:text -> a description of the boat.
    • captain:string -> the name of the captain operating the boat.
  • Species
    • short_name:string -> a three character abbrevation of the species.
    • name:string -> the species name.
  • QR
    • trip:integer -> foreign key reference to a trip.
    • species:integer -> foreign key reference to a species.
